## 🚀 今日工作详述

### 1. 新增与封装核心 UI 组件 (`ProjectCard` & `ArticleCard`)
* [cite_start]**实现 UI 与逻辑解耦**：将原本散落在各个路由页面（如 `index.astro`）中的卡片 HTML 标签与局部 CSS 样式提取，在 `src/components/` 目录下封装了独立的 `ProjectCard.astro` 和 `ArticleCard.astro` 组件，大幅精简了页面代码量 [cite: 1]。
* **引入变体设计模式 (Variant Pattern)**：
    * [cite_start]在组件的 `Props` 接口中新增了 `category` 属性作为“皮肤指令”接收口 [cite: 1]。
    * [cite_start]利用动态类名拼接技术（如 `` <a class={`project-card ${category}`}> ``），实现了**“一套骨架，多种视觉皮肤”**的架构 [cite: 1]。
    * [cite_start]在组件内部独立维护了不同合集的专属视觉规范（如为 `analog_ic` 配置橙色系，为 `rfic` 预留蓝色系），实现了极高的扩展性 [cite: 1]。

### 2. 博客组织结构深度重构：引入 `intro` (主博客) 概念
* [cite_start]**松绑数据校验限制**：在 `src/content.config.ts` 中，将项目级元数据（`projectName` 和 `projectDescription`）修改为 `.optional()` 可选字段 [cite: 1]。
* [cite_start]**彻底贯彻 DRY 原则**：重构了 Markdown 元数据标准。现在，同一项目下的信息只需集中在第一篇介绍性博客（如 `intro.md`）中编写，其余子模块博客保持极简结构，消灭冗余 [cite: 1]。
* **路由大脑控制室“智能寻宝”与安全加固**：
    * [cite_start]重构了**一级大门** (`index.astro`) 和**二级动态枢纽** (`[project]/index.astro`) 的底层数据流 [cite: 1]。
    * [cite_start]通过编写 JavaScript 高阶数组方法 (`.filter()` 与 `.find()`)，让系统自动在项目列表中精准抓取带有介绍信息的 `intro` 文章 [cite: 1]。
    * [cite_start]增加了健壮的**三元运算符兜底机制**，即使暂未撰写介绍文章，系统依然会使用底层 `projectId` 渲染，保障程序坚不可摧 [cite: 1]。
